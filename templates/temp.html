Function Overloading

Using the same function name for different operations

Function Overloading

Function overloading allows multiple functions to have the same name, as long as their parameters are different in type or number. This lets you use the same function name for similar tasks.

Key Concept

Overloaded functions must differ in their parameter types or number of parameters. The return type alone is not sufficient to differentiate functions.

Without Function Overloading

Consider this example with two functions that add numbers of different types:

int plusFuncInt(int x, int y) {
    return x + y;
}

double plusFuncDouble(double x, double y) {
    return x + y;
}

int main() {
    int myNum1 = plusFuncInt(8, 5);
    double myNum2 = plusFuncDouble(4.3, 6.26);

    cout << "Int: " << myNum1 << endl;
    cout << "Double: " << myNum2;
    return 0;
}
Problem

We had to create two different function names for essentially the same logic. This makes the API harder to remember and use.

Using Function Overloading

Instead of defining two functions with different names, we can overload one function name to work for both int and double types:

int plusFunc(int x, int y) {
    return x + y;
}

double plusFunc(double x, double y) {
    return x + y;
}

int main() {
    int myNum1 = plusFunc(8, 5);
    double myNum2 = plusFunc(4.3, 6.26);

    cout << "Int: " << myNum1 << endl;
    cout << "Double: " << myNum2;
    return 0;
}
Important Note

Multiple functions can have the same name as long as the number and/or type of parameters are different. The compiler selects the appropriate function based on the arguments provided.

Overloading by Number of Parameters

You can also overload functions by using a different number of parameters:

int plusFunc(int x, int y) {
    return x + y;
}

int plusFunc(int x, int y, int z) {
    return x + y + z;
}

int main() {
    int result1 = plusFunc(3, 7);
    int result2 = plusFunc(1, 2, 3);

    cout << "Sum of 2 numbers: " << result1 << endl;
    cout << "Sum of 3 numbers: " << result2;
    return 0;
}

Key Takeaways

  • Function overloading allows multiple functions with the same name but different parameters
  • Parameters must differ in type, number, or both to allow overloading
  • Return type alone is not sufficient to differentiate overloaded functions
  • Overloading makes APIs more intuitive and reduces the need for unique function names
  • Compiler selects the appropriate function based on arguments provided
  • Overloading improves code readability and maintainability
